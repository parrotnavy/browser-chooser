5MB未満 • ネイティブ性能

選ばれる理由

🎯

スマートルーティング

リンクの行き先を細かく定義できます。URLパターン、起動元アプリ、修飾キーで振り分け可能。仕事用はWork Chrome、 個人用はSafariへ。

🔀

マルチプロファイル

ウィンドウの切り替えに悩みません。Chrome、Edge、Brave、Vivaldi、Arcのプロファイルを自動検出し、 それぞれを個別ターゲットとして使えます。

🛡️

プライバシー優先

リンクを自動でクリーン化。utm_source、fbclid、gclid などの 追跡パラメータをブラウザーに渡す前に削除します。

🎨

使いやすく美しいUI

ルールが一致しない場合も快適に選択できます。RowレイアウトとRadialメニューを用意しています。

⚡

ネイティブの速さ

Swift 6(macOS) と .NET 9(Windows) でネイティブ実装。起動が速く、メモリ使用量も最小限です。

🪶

軽量設計

5MB未満。不要な依存関係を省き、必要な性能に集中しました。

🔗

URL展開

移動前にリンク先を確認できます。bit.ly や t.co などの短縮URLを自動展開し、正確な振り分けを実現します。

⚙️

ルールエンジン

"contains"、"regex"、"ends with" に対応。上から順に評価され、より具体的なルールが優先されます。

📡

開発者フレンドリー

カスタムURLスキームAPI (x-browser-chooser://) で、スクリプトから特定ブラウザーを直接起動できます。

よくある質問

なぜ Browser Chooser はこんなに軽いのですか？

Browser Chooser は Swift(macOS) と .NET(Windows) でネイティブ実装されています。Chromium同梱やNode.jsランタイムが 不要なため、5MB未満で高速起動・低メモリを実現しています。

Browser Chooser は無料ですか？

はい、無料です。サブスクなし、トラッキングなしで、より良いブラウジング体験を提供します。

データはどのように扱われますか？

プライバシーを最優先しています。URL処理とルール評価はすべてローカルで実行され、閲覧履歴を収集・保存・送信しません。

Windows でも使えますか？

はい。多くのURLルーターがmacOS限定の中、Browser ChooserはWindows 10/11向けにもネイティブ体験を提供します。
