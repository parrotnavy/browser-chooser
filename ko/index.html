Browser Chooser로 링크 흐름을 제어하세요.
업무 링크는 Chrome으로, 개인 링크는 Safari로, 추적 파라미터는 자동 제거합니다.

5MB 미만 • 네이티브 성능

이런 점이 좋습니다

🎯

스마트 라우팅

링크 목적지를 정확히 정의하세요. URL 패턴, 실행 앱, 보조 키 기준으로 분기할 수 있습니다. 업무 링크는 Work Chrome, 개인 링크는 Safari로 보낼 수 있습니다.

🔀

멀티 프로필

창 전환에 시간을 쓰지 마세요. Chrome, Edge, Brave, Vivaldi, Arc 프로필을 자동 인식하며 각 프로필을 개별 대상으로 사용합니다.

🛡️

프라이버시 우선

링크를 자동 정리합니다. utm_source, fbclid, gclid 같은 추적 파라미터를 브라우저로 전달하기 전에 제거합니다.

🎨

직관적이고 아름다운 UI

규칙이 일치하지 않을 때도 깔끔하게 선택하세요. Row 레이아웃과 Radial 메뉴를 지원합니다.

⚡

네이티브 속도

Swift 6(macOS)와 .NET 9(Windows)로 네이티브 구현했습니다. 즉시 실행되고 메모리 사용이 적으며 OS와 잘 어울립니다.

🪶

작은 용량

5MB 미만의 가벼운 앱입니다. 불필요한 의존성 없이 핵심 성능에 집중했습니다.

🔗

URL 확장

이동 전에 목적지를 확인하세요. bit.ly, t.co 같은 단축 URL을 자동 확장해 더 정확한 라우팅을 지원합니다.

⚙️

규칙 엔진

"contains", "regex", "ends with" 조건을 지원합니다. 위에서 아래로 평가되어 더 구체적인 규칙이 우선 적용됩니다.

📡

개발 친화적

커스텀 URL 스킴 API(x-browser-chooser://)를 사용해 스크립트에서 특정 브라우저를 직접 열 수 있습니다.

자주 묻는 질문

Browser Chooser는 왜 이렇게 작나요?

Browser Chooser는 Swift(macOS)와 .NET(Windows) 기반의 네이티브 앱입니다. Chromium 번들, Node.js 런타임이 없어 5MB 미만 용량으로 빠르게 시작하고 메모리도 적게 사용합니다.

Browser Chooser는 무료인가요?

네, 무료로 사용할 수 있습니다. 구독도 없고 추적도 없습니다. 더 나은 브라우징 경험만 제공합니다.

내 데이터는 어떻게 처리되나요?

프라이버시를 최우선으로 설계했습니다. URL 처리와 규칙 평가는 모두 로컬에서 수행되며, 방문 기록을 수집·저장·전송하지 않습니다.

Windows에서도 사용할 수 있나요?

네. 많은 URL 라우터가 macOS 전용인 반면, Browser Chooser는 Windows 10/11 사용자에게도 네이티브 경험을 제공합니다.
